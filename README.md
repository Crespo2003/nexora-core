# Nexora Core

Nexora Core is the Sprint 001 foundation plus the Sprint 002 AI Document Centre for a Malaysia-first AI Real Estate Operating System. This build focuses on the Rental Command Centre, TA Memory, rental collections, bilingual UI, Supabase persistence, AI Tenancy Import v1, and secure document review workflows.

## Local Setup

```bash
pnpm install
pnpm test
pnpm build
pnpm dev
```

## Environment Variables

Client-safe variables:

```bash
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
```

The Sprint 001 server routes use the anon/publishable key so Supabase RLS and Storage policies remain the enforcement layer. Do not expose or configure privileged database keys for this browser-facing app unless authentication and authorization are added first.

## Supabase Setup

Run all migrations in `supabase/migrations` against the target Supabase project. Sprint 001 requires:

- `tenancies`
- `rental_collections`
- `tenancy_documents`
- `tenancy_extractions`
- private Storage bucket `tenancy-documents`

The corrective collection migration keeps `total_due` and `outstanding_balance` compatible with both generated-column schemas and older development schemas. The app does not insert or update generated columns directly.

Sprint 002 adds:

- `documents`
- `document_extractions`
- `document_links`
- `import_jobs`
- `document_activity`
- private Storage bucket `real-estate-documents`

The Sprint 002 migration is idempotent and includes indexes, `updated_at` triggers, RLS, beta permissions, private bucket creation, and Storage policies.

## Storage Bucket

The Sprint 001 migration creates or updates the private bucket:

```text
tenancy-documents
```

Allowed file types:

- PDF
- DOCX

Maximum file size:

- 10 MB

Documents are viewed through short-lived signed URLs generated server-side.

Sprint 002 creates or updates the private bucket:

```text
real-estate-documents
```

Allowed file types:

- PDF
- DOCX
- JPG / JPEG
- PNG

Maximum file size:

- 10 MB

Document Centre files are viewed and downloaded through 5-minute signed URLs generated by server routes.

## AI Tenancy Import v1

Supported:

- Text-based PDF
- DOCX
- Deterministic fallback extraction
- Editable review before import
- Confidence display
- Supabase document metadata
- Private document upload

Not included in v1:

- Scanned-image OCR
- Advanced AI extraction without a configured server-side AI provider

If advanced AI extraction is added later, use server-only credentials. Do not expose AI keys to the browser.

## AI Document Centre

Route:

```text
/documents
```

Supported:

- Document dashboard metrics
- PDF, DOCX, JPG, JPEG, and PNG upload queue
- Drag-and-drop and browse upload
- Duplicate filename warning
- Document type selection
- Private Supabase Storage upload
- Classification, parsing, extraction, and review stages
- Editable tenancy review fields
- Save draft in localStorage
- Confirm import to create tenancy, initial collection, document link, and import job
- Secure view and download buttons using signed URLs

OCR status:

- Image/scanned files are detected and marked as OCR required.
- A provider interface and fallback implementation exist.
- No third-party OCR provider is configured by default.

## Testing Checklist

- Create tenancy and confirm initial collection is also created.
- Edit tenancy and refresh to confirm persistence.
- Delete tenancy and confirm linked collections are removed.
- Record payment and confirm payment status updates.
- Upload text-based PDF tenancy agreement.
- Upload DOCX tenancy agreement.
- Review extracted data and correct fields.
- Confirm AI import and verify tenancy, document, extraction, and collection records.
- Retry a failed import without re-uploading the file.
- Open uploaded document through the View Document button.
- Switch EN / 中文 and confirm immediate language change.
- Refresh and confirm language persistence.
- Verify all actual dates display as `DD/MM/YYYY`.
- Verify collection months display as `MM/YYYY`.
- Confirm no success message appears when a required write fails.
- Open `/documents` and verify the Document Centre navigation entry.
- Upload PDF, DOCX, JPG/JPEG, and PNG files.
- Confirm image/scanned files show OCR-required messaging instead of invented extracted fields.
- Save a review draft, refresh, and confirm the draft reloads.
- Confirm import creates the tenancy, initial rental collection, document link, and completed import job.
- Confirm duplicate imports show a review-required warning and do not merge records automatically.
- View and download uploaded files through signed URLs.

## Deployment Notes

- Apply migrations before deploying the app.
- Confirm RLS policies and grants exist for all Sprint 001 tables.
- Confirm Storage policies exist for `tenancy-documents`.
- Confirm RLS policies and grants exist for all Sprint 002 document tables.
- Confirm Storage policies exist for `real-estate-documents`.
- Keep the bucket private.
- Configure server-only secrets in the deployment platform, never in source control.
